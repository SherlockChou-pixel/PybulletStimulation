### 1.4 当前阶段结论

#### 已经证实的内容

- 系统主流程已经稳定，具备毕设演示基础
- 视觉定位精度稳定在毫米级
- 多模态状态评估与日志链路完整
- 闭环控制会频繁介入，说明它确实参与了系统稳定化

#### 当前还没有被“实验充分证明”的内容

- 关闭闭环后，最终成功率仍然是 100%，说明当前场景难度偏低
- 关闭回正后，最终成功率仍然是 100%，说明当前场景下 recenter 的收益还不够明显
- 真正暴露风险的是搬运/放置阶段，而不是抓取瞬间

#### 当前最重要的问题

> 现有实验可以证明“系统能稳定运行”，但还不足以强力证明“多模态融合 + 闭环控制 + 回正机制显著优于对照方法”。

## 3. 下一轮实验方案

## 第一阶段：先做可直接执行的统计增强实验

这一步**不改核心算法**，直接利用现有脚本增加样本量。

### 3.1 实验组设置

#### 组 A：基线组

```bash
python scripts/batch_experiments.py --runs 50 --output logs/round2/baseline_50.csv --log-file logs/round2/baseline_50.log
```

#### 组 B：无闭环组

```bash
python scripts/batch_experiments.py --runs 50 --disable-closed-loop --output logs/round2/no_closed_loop_50.csv --log-file logs/round2/no_closed_loop_50.log
```

#### 组 C：无回正组

```bash
python scripts/batch_experiments.py --runs 50 --disable-recenter --output logs/round2/no_recenter_50.csv --log-file logs/round2/no_recenter_50.log
```

### 3.2 统计指标

对每组统一统计：

- 抓取成功率
- 放置成功率
- 重试率
- 平均尝试次数
- 平均闭环触发次数
- 平均回正次数
- 抬升后抓取置信度
- 释放前抓取置信度
- 抬升后滑移风险
- 释放前滑移风险
- 平均仿真时长

### 3.3 这一阶段的目标

- 先确认现有结论是否稳定
- 判断“无闭环/无回正”是否在样本量增加后出现明显退化
- 为后续困难场景实验提供对照基线

## 第二阶段：引入困难场景，把差距拉开

这一阶段是下一轮实验的重点。

### 3.4 需要先补的小功能

建议先补充批量实验脚本参数，至少支持：

- 自定义物体生成范围
- 自定义物体质量
- 自定义摩擦系数
- 批量切换物体 URDF
- 为实验结果增加 `scenario_name`

### 3.5 建议构造的困难场景

建议至少做 4 类：

#### 场景 S1：基础随机场景
- 作为对照基准
- 与当前实验一致

#### 场景 S2：边缘可达场景
- 物体放在工作空间边缘
- 目标：放大位姿候选与初始抓取不准的问题

#### 场景 S3：低摩擦场景
- 降低物体或接触面的摩擦系数
- 目标：放大滑移风险，检验压力+惯性+闭环的作用

#### 场景 S4：高质量/重物体场景
- 增大物体质量
- 目标：放大搬运和放置阶段的不稳定性

### 3.6 每个场景下的实验组

每个困难场景都做 3 组：

1. 基线
2. 无闭环
3. 无回正

建议每组 30 次。

总规模建议：

- 4 个场景 × 3 个策略 × 30 次 = 360 次

如果时间紧，可先做：

- 3 个场景 × 3 个策略 × 20 次 = 180 次

### 3.7 这一阶段预期输出

这一阶段最希望看到：

- 基线组成功率高于无闭环
- 基线组释放前滑移风险低于无闭环
- 基线组重试率低于无回正
- 困难场景下对照组失败开始出现，基线更稳

## 第三阶段：多模态贡献实验

这一步用于支撑“多模态”而不仅是“闭环”。

### 3.8 建议新增的消融组

这部分需要增加开关后再做：

1. 仅视觉
2. 视觉 + 压力
3. 视觉 + 压力 + 惯性（完整方法）

### 3.9 目标

验证：

- 只用视觉时，无法有效识别接触稳定性
- 加入压力后，抓取质量判断更可靠
- 再加入惯性后，搬运阶段的风险识别更好

### 3.10 建议优先级

如果时间有限：

- **优先先做闭环/回正对照**
- 多模态消融放在第二优先级
